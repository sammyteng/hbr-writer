# 骨架一：领导力洞察

适用场景：对领导力议题给出框架化、有案例支撑的管理洞察。

## 结构模板

```
【核心论题（反转现有认知）】
「大多数[管理者/领导者]相信[常见观点]。但研究表明，这是错的。」
用一个反直觉的核心主张开场。

【「领导力框架」命名与定义】
给文章的核心框架命名（这是 HBR 的标志性操作）：
例：「我们称之为[框架名称]」
框架通常包含：
- 2个维度 → 2×2矩阵
- 3个阶段 → 发展路径模型
- 4种类型 → 领导者类型分类

【案例支撑 × 2-3个】
每个框架维度配一个企业案例：
「在[公司]，当[CEO/高管]面临[场景]时，他们[选择A]，结果[X]。」
案例规模：优先全球知名 > 中国标杆 > 匿名案例

【可操作建议：你应该怎么做】
给管理者带走的「今日可用」行动指南：
第一步：[具体行动]
第二步：[具体行动]
第三步：[具体行动]
（每步避免「加强沟通」等空洞建议）

【挑战读者的反思问题】
「想想你自己的团队：[具体反思场景]。你的答案是什么？」
```

# 骨架二：战略决策框架

适用场景：为高管提供面对战略决策难题的框架化思考工具。

## 结构模板

```
【战略困境的典型场景】
「当[战略场景]出现时，大多数管理者的本能选择是[A]。但这恰恰是问题所在。」

【决策框架建立】
提供战略决策的思考模型：
- 框架名称：[X矩阵 / Y原则 / Z模型]
- 框架维度定义
- 如何使用这个框架做出更好决策

【历史案例验证】
[公司A] 使用了框架中的[X策略]，结果：[数据]
[公司B] 忽视了[Y原则]，结果：[后果]

【执行中的常见陷阱】
实施这个框架时，管理者最常犯的3个错误：
① [陷阱1] + 如何避免
② [陷阱2] + 如何避免
③ [陷阱3] + 如何避免

【结尾：框架的适用边界】
诚实说明框架的局限性：
「这个框架在[场景]中非常有效，但在[例外场景]中需要调整。」
```

# 骨架三：创新与变革管理

适用场景：帮助企业领导者理解和推动组织内部的创新与变革。

## 结构模板

```
【变革为什么总是失败】
用数据开篇：「研究显示，[X%]的企业变革计划以失败告终。原因不是变革方向错了，而是[根本原因]。」

【成功变革的「关键杠杆点」框架】
提供推动变革的框架：
- 感知紧迫感（Urgency）
- 组建引导联盟（Coalition）
- 设计愿景和策略（Vision）
- 赋能广泛行动（Empowerment）
（可引用科特尔8步变革模型等经典框架，或提出新的本土化版本）

【中国企业案例】
选2个本土变革成功/失败案例：
- 成功案例：[华为/海尔/美的]的[变革举措]如何实现
- 失败案例：[某知名公司]的[变革尝试]为何受阻

【个人角色：中层管理者如何推动变革】
HBR 特色：不只写 CEO 视角，也写「夹心层」的实操：
- 如何在没有授权的情况下推动变革？
- 如何管理变革疲劳？
```

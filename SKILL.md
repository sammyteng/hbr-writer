# 哈佛商业评论（HBR）风格写作 Skill

## 账号画像

**哈佛商业评论中文版** — 全球顶尖管理思维平台
- 核心定位：为商业领袖提供最前沿的管理理论与实践指导
- 受众：企业高管、管理者、MBA学生、商学院教授
- 调性：**权威 + 框架化 + 案例驱动 + 实践导向**
- 引用大师：德鲁克、波特、克里斯坦森、科特尔

---

## 核心写作特征

### 1. 标题公式

```
【框架型】
→ "成为[目标]的[N]个关键原则"
→ "为什么[普遍做法]是错的：[更好方案]"
→ "[管理难题]，正确的做法是[方法论]"

【反直觉型】
→ "优秀[管理者/领导者]不会告诉你的[N]件事"
→ "为什么[常见共识]，在[场景]下反而有害"

【情境应用型】
→ "面对[商业场景]，[职级/角色]应该这样做"
→ "[行业变局]下的领导力：[N]个关键转变"
```

### 2. 文章结构

```
【核心观点陈述（「论题」先行）】
HBR文章通常第一段就点明核心管理洞察：
- 「[普遍做法]——但这个假设是错的。研究表明，[反转]。」
- 「当[场景]发生时，大多数管理者的本能反应是[A]，但真正有效的做法是[B]。」

【理论框架建立】
提供一个「管理框架」或「决策模型」：
- 通常是2×2矩阵、四步模型、三层结构等
- 框架要有命名（让读者可以带走）
- 框架的每个维度都要清晰定义

【案例支撑层】
每个核心观点都用2-3个企业案例支撑：
- 优先选全球知名公司（亚马逊/谷歌/苹果/宝洁/GE）
- 也可以有中国本土案例（阿里/华为/海尔）
- 案例要聚焦「决策时刻」而非公司全貌

【可操作建议（「所以你应该…」）】
HBR文章必须落到「对读者有什么用」：
- 给出至少3条具体、可操作的管理建议
- 每条建议配合一个「实施要点」
- 使用「第一步…第二步…」等清晰结构

【结尾：挑战读者认知】
- 用一个认知挑战或反向提问结尾
- 「想想你自己的团队：[反思问题]？」
```

### 3. 写作风格原则

**必须做到**：
- 每篇文章必须有1个清晰可命名的「管理框架」
- 理论主张必须有学术研究或大量案例支撑
- 文章末尾必须有「可操作的管理建议」
- 语气专业但不艰涩（对高管级别读者）

**绝对不能**：
- 只有观点没有框架（HBR的核心是「方法论」）
- 案例太少或太泛（要有具体的公司决策细节）
- 建议太抽象（「更好地沟通」这类无意义建议）
- 忽视中国商业实践（中文版要结合中国场景）

## STEP 流程

### STEP 1：确认参数
```
1. 管理议题：领导力 / 战略 / 创新 / 人才 / 组织变革 / 数字化？
2. 目标读者层级：基层管理者 / 中层 / C-suite高管？
3. 核心「管理洞察」是什么（用一句话描述你想传达的方法论）？
4. 字数：1500 / 2500 / 3500字
```

### STEP 2：设计「管理框架」
在写作前，先设计文章的核心框架：
- 框架名称是什么？
- 框架有几个维度/步骤/象限？
- 每个维度的定义是什么？

### STEP 3：案例采集
重点渠道：
- HBR历史案例数据库（hbr.org）
- 著名企业年报/管理层访谈
- 商学院案例研究（哈佛/沃顿/MIT Sloan）
- 管理大师著作（《创新者的窘境》《从优秀到卓越》等）

### STEP 4：按骨架写正文

### STEP 5：自检清单
```
□ 第一段是否清晰陈述了核心管理洞察（不是铺垫）？
□ 文章是否有一个清晰可命名的「管理框架」？
□ 每个核心观点是否有至少2个具体企业案例支撑？
□ 文章末尾是否有≥3条可操作的管理建议？
□ 建议是否具体（避免「加强沟通」这类空话）？
□ 是否有结合中国商业实践的本地化内容？
```
